Core Idea

Rear pointer wraps back to front via % capacity — empty dequeued slots are recycled.

4 ADT Operations

enqueue(item) O(1)
dequeue() O(1)
is_empty() O(1)
__len__() O(1)

Implementation

Uses collections.deque internally. Capacity is optional — bounded for wheel slots, unbounded for task buckets.

Circular Queue — Applications

🖥️

CPU Round-Robin Scheduling

OS cycles through processes in fixed time slices — wraps back to the first process automatically.

🌐

Network Packet Buffers

Routers store incoming packets in a ring buffer — new packets overwrite oldest if buffer is full.

🎵

Audio Streaming Buffers

Producer (decoder) and consumer (speaker) share a ring buffer — avoids blocking and dropped frames.

Algorithm

Hierarchical Timing Wheel

Two-level wheel structure

How It Works

  • 1.Call schedule(item, delay_s)
  • 2.delay < 60s → second wheel slot (hand+delay)%60
  • 3.delay ≥ 60s → minute wheel slot (hand+delay÷60)%60
  • 4._tick() fires tasks in current second bucket
  • 5.On full rotation → _cascade_minute_slot() moves minute tasks down

Real-World Usage

Linux kernel timer Apache Kafka DelayQueue Network timeout mgmt

Key Edge Case

System idle for 90s? advance_to_now() uses an elapsed-tick loop to replay all missed seconds — we got this wrong the first time.

Lower bar = fewer operations per insert

Our Timing Wheel — per operation

Operation Complexity Notes
schedule() O(1) Slot via modulo
_tick() O(k) k tasks in that bucket
_cascade() O(m) m tasks; once/minute
advance_to_now() O(elapsed) Indep. of total timers

Key insight: Cost depends only on tasks due now, not total timers in the system. A busy system with 100,000 pending timers costs the same as one with 10.
